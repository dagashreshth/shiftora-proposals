Platforms & Applied AI

1 system in production
UAE travel group: visa platform & travel assistant
UAE · longest partnership

Visa processing was manual and document-heavy, dying on errors nobody caught until submission. Support could not reason across a customer’s own passport and visa history.

What we built
  • Document intelligence agent: OCR with confidence-routed extraction; low-confidence fields route to review, and nothing submits until it validates against country rules.
  • Visa workflow agent: a versioned rule engine per route and visa type; rule changes ship as versions, so past decisions stay reproducible.
  • Travel assistant: books, plans, and self-corrects with live context on visa and passport validity.
  • Platform: multi-tenant MongoDB with EntityId scoping on every query path; .NET Minimal API core; Next.js and React Native surfaces.

The backtest protocol, published in full
  • Holdout, frozen at kickoff. The most recent 12 months of history are sealed before any model work begins; nothing ever trains on them.
  • Rolling-origin evaluation. Each holdout month is forecast from an origin that sees only earlier data; the origin rolls forward and repeats. Twelve origins, twelve out-of-sample tests.
  • Accuracy and bias, gated together. Accuracy is reported as 100 minus WAPE; bias is gated at plus or minus 3 percent alongside it, so accuracy cannot be achieved through systematic over- or under-forecasting.
  • Baseline first. The incumbent method is scored under the identical protocol before ours; the claim is the difference, same data, same window, same math.
  • Re-proven every cycle. The backtest re-runs on every retrain and rule change. Any figure that fails re-validation is withdrawn.

The three things a platform build actually risks

The engineering risk in productising IP is not the idea; it is tenant isolation (one client’s data never touches another’s), versioning (a method that ships, diffs, and rolls back like software), and evaluation discipline (a module that must pass its golden set to deploy). These are the three things every system in 01 already demonstrates, and the reason a methodology firm should not rebuild them from scratch.
